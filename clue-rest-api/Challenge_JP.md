# 🛠 CLUe API テクニカルサポートアセスメント

CLUe API テクニカルサポート統合テストへようこそ。このアセスメントは、APIドキュメントの理解、Postmanでの環境ワークフローの設定、エンドユーザー向けの技術プロセスの文書化能力を評価します。

## 🎯 目的
1. **APIの理解**: CLUe APIドキュメントをナビゲートし、機能的なリクエストを実行する能力を実証する
2. **Postmanのマスタリー**: 環境、変数、コレクションを効果的に使用する
3. **ドキュメンテーション**: Postmanコレクション内に明確な「サポート準備完了」の指示を提供する

---

## 📖 タスク
CLUe APIを使用して特定のユーザー管理ワークフローを実行する**Postmanコレクション**を構築する必要があります。物理デバイスをお持ちでないため、**サーバー/クラウド**ロジックに焦点を当ててください。

### ドキュメンテーションリファレンス
[CLUe APIガイド](https://twilight-value-ef0.notion.site/CLUe-API-Guide-11f03724c781804a9682c27c9fe331d9)

### 🔑 認証に関する注意
このテストを実行するには、**APIキー**と**Place ID**が必要です。
* テストを開始する前に、連絡担当者に連絡して一時的な資格情報をリクエストしてください。
* 受け取ったら、セキュリティのためにPostman変数を使用してこのキーを実装する必要があります。

### シナリオ
顧客がユーザー管理を自動化する必要があります。次の**4つのステップ**を実行するコレクションを作成する必要があります：

1. **システムチェック**: 接続を確認するために、すべての「Doors」または「Places」のリストを取得する
2. **ユーザー作成**: 一意のIDと名前を持つ新しいユーザーを作成する
3. **ステータス更新**: その特定のユーザーのステータスを`INACTIVE`に更新する
4. **ユーザー削除**: 作成したユーザーを削除する
5. **ログ取得**: 特定の時間枠に関連するイベントを見つけるためにシステムログを照会する（ログを取得するには以下のエンドポイントを使用）
```code=api
https://api.moon.supremainc.com/v1/audit-logs?size={{size}}&placeId={{placeId}}&startAt={{Time in utc epoch milliseconds }}&endAt={{Time in utc epoch milliseconds}}
````

## 🛠 技術要件

### 1. Postman環境
URLやボディに値（IDやキーなど）をハードコードしないでください。次の内容を含むPostman環境ファイルを使用する必要があります：
* `baseUrl`
* `apiKey`
* `placeId`
* `authId`（これは「ユーザー作成」リクエストの応答からキャプチャする必要があります）

### 2. ドキュメンテーション
Postmanコレクション内で：
* すべてのリクエストに、リクエストの説明を記載する必要があります
* 説明には、顧客向けの「プロのヒント」を強調表示するためにMarkdownを使用してください

### 3. 検証（テスト）
各リクエストには、成功を検証するための少なくとも1つの基本的なテストを**テスト**タブに含める必要があります（例：`200 OK`または`201 Created`ステータスコードを確認する）

## 📤 提出方法
次の内容を含む圧縮（`.zip`）フォルダーを提供してください：
1. エクスポートされた**Postmanコレクション**（v2.1）
2. エクスポートされた**Postman環境**（ダミー値または試用値を含む）
3. コレクションのインポートと実行方法に関する簡単な指示ノート

**評価基準**：
* **正確性**：リクエストはドキュメントに従っていますか？
* **組織**：コレクションは読みやすく使いやすいですか？
* **変数**：`{{variable_name}}`構文を正しく使用しましたか？
